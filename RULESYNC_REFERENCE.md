# Rulesync Reference for AI Agents

This file contains links and instructions for validating and working with rulesync configuration in this repository.

## Documentation Sources

### Local (fastest, always check first)

- `.claude/skills/rulesync/` — full rulesync documentation cached locally:
    - `file-formats.md` — file format specifications for rules, commands, subagents, skills, hooks, mcp, aiignore
    - `configuration.md` — rulesync.jsonc configuration reference
    - `cli-commands.md` — CLI commands (generate, init, check, etc.)
    - `supported-tools.md` — list of all supported AI tools
    - `declarative-sources.md` — how to import rules from other repos
    - `global-mode.md` — global vs project-level installation
    - `simulated-features.md` — simulating commands/subagents/skills as rules

### Remote (for latest version)

- GitHub repo: `https://github.com/dyoshikawa/rulesync`
- File formats doc: `https://raw.githubusercontent.com/dyoshikawa/rulesync/main/docs/reference/file-formats.md`
- Releases page: `https://github.com/dyoshikawa/rulesync/releases`

### JSON Schemas (for validation)

- Config schema: `https://github.com/dyoshikawa/rulesync/releases/latest/download/config-schema.json`
- MCP schema: `https://github.com/dyoshikawa/rulesync/releases/latest/download/mcp-schema.json`
- Both schemas are referenced via `$schema` field in `rulesync.jsonc` and `.rulesync/mcp.json`

### Repomix-toon (rulesync source analysis)

Download: `https://github.com/dyoshikawa/rulesync/releases/latest/download/repomix-toon.zip`

Contains .toon files for deep analysis of rulesync internals. Key files:

- `repomix-output-features-rules.toon` — how rules are generated
- `repomix-output-features-skills.toon` — how skills are generated
- `repomix-output-features-subagents.toon` — how subagents are generated
- `repomix-output-features-commands.toon` — how commands are generated
- `repomix-output-features-hooks.toon` — how hooks are generated
- `repomix-output-features-mcp.toon` — how MCP config is generated
- `repomix-output-configs.toon` — config parsing and validation
- `repomix-output-types.toon` — TypeScript type definitions (useful for understanding schemas)
- `repomix-output-full.toon` — complete codebase (4.3MB, use specific files instead)

To download and use:

```bash
curl -sL "https://github.com/dyoshikawa/rulesync/releases/latest/download/repomix-toon.zip" -o /tmp/repomix-toon.zip
unzip -o /tmp/repomix-toon.zip -d /tmp/repomix-toon
# Then read specific .toon files as needed
```

## How to Validate This Repository

### Quick validation

```bash
rulesync check
```

### Manual validation checklist

1. `rulesync.jsonc` — validate against config-schema.json
2. `.rulesync/rules/*.md` — each must have frontmatter: root, targets, description, globs
3. `.rulesync/subagents/*.md` — each must have frontmatter: name, targets, description, claudecode.model, claudecode.tools
4. `.rulesync/skills/*/SKILL.md` — each must have frontmatter: name, description, targets (claudecode block optional)
5. `.rulesync/commands/*.md` — each must have frontmatter: description, targets
6. `.rulesync/hooks.json` — must have `"version": 1` if present
7. `.rulesync/mcp.json` — validate against mcp-schema.json
8. `.rulesync/.aiignore` — gitignore-style patterns

### Download schemas for offline validation

```bash
curl -sL "https://github.com/dyoshikawa/rulesync/releases/latest/download/config-schema.json" -o /tmp/config-schema.json
curl -sL "https://github.com/dyoshikawa/rulesync/releases/latest/download/mcp-schema.json" -o /tmp/mcp-schema.json
```

## Project Structure

```
rulesync.jsonc                    # Main config — targets, features, options
.rulesync/
  .aiignore                       # Files to exclude from AI analysis
  hooks.json                      # Lifecycle hooks (sessionStart, postToolUse, stop, etc.)
  mcp.json                        # MCP server definitions
  rules/                          # Development rules (*.md with frontmatter)
    overview.md                   # Root rule (root: true) — becomes CLAUDE.md
    *.md                          # Non-root rules — become .claude/rules/*.md
  subagents/                      # Agent definitions (*.md with frontmatter)
  skills/                         # Skill directories (each has SKILL.md)
  commands/                       # Slash commands (*.md with frontmatter)
  hooks/                          # Hook scripts referenced by hooks.json
.claude/                          # Generated by rulesync (mostly gitignored)
  settings.json                   # Permissions + hooks config
  settings.local.json             # Local overrides (gitignored)
```
